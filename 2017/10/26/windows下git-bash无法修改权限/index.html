windows下git-bash无法修改权限

前天在windows10下面打开git bash命令行，创建一个脚本hexo.sh发现执行以下指令：

之后使用ls -al hexo.sh查看文件的权限时发现并没有可执行权限。

故google之。

解决帖

主要原因

因为windows使用的是NTFS或者FAT32文件系统，所以并不支持像unix系统那样可以修改可执行权限的操作。

windows下主要是根据文件名字或者内容来判断是否可执行。

  1. 常用的可执行文件的后缀有：.com、.bat、.exe
  2. 文件中以#!开头，例如#!/bin/bash

在windows下的git bash下面如果一个文件是可执行的，可以看到该文件名是绿色，并且后面还会加上一个*号.

设置好hexo.sh脚本之后就可以简化使用hexo发表博客的流程了。
